HTML Practice — Understanding Semantic HTML

In this lesson, you’ll learn about semantic HTML tags that describe the meaning and structure of a webpage, not just how it looks. These tags make pages easier for browsers, search engines, and assistive technologies to understand.

What is Semantic HTML?

Semantic HTML uses meaningful tags to define different parts of a webpage. Instead of using only <div> and <span>, semantic tags like <header>, <footer>, and <article> tell what kind of content they contain.

For example, a <header> represents the top section of a page, while an <article> represents a standalone piece of content like a blog post or a news story.

Common Semantic Tags

<header>

The header element usually contains a title, logo, or navigation menu.

Example:

<header>
  <h1>My Website</h1>
  <nav>
    <a href="#home">Home</a> |
    <a href="#about">About</a> |
    <a href="#contact">Contact</a>
  </nav>
</header>

<nav>

The navigation element defines a set of links for navigating the website.

Example:

<nav>
  <a href="#news">News</a> |
  <a href="#articles">Articles</a> |
  <a href="#contact">Contact</a>
</nav>

<section>

The section element groups related content together. Each section should have its own heading.

Example:

<section>
  <h2>About Us</h2>
  <p>We are a team of learners exploring HTML.</p>
</section>

<article>

The article element is used for independent, self-contained content such as a blog post, news story, or forum post.

Example:

<article>
  <h2>HTML Becomes Easier</h2>
  <p>With practice, HTML becomes simple to understand.</p>
</article>

<aside>

The aside element contains content related to the main content but not essential to it. It’s often used for sidebars or notes.

Example:

<aside>
  <h3>Tip</h3>
  <p>Keep your code clean and properly indented!</p>
</aside>

<figure> and <figcaption>

These tags are used for images, diagrams, or code examples with captions.

Example:

<figure>
  <img src="semantic-structure.png" alt="Semantic HTML layout" width="400">
  <figcaption>Figure 1: Example of semantic HTML layout.</figcaption>
</figure>

<footer>

The footer element usually contains author information, copyright, or related links at the bottom of the page.

Example:

<footer>
  <p>&copy; 2025 My Website</p>
</footer>

Putting It All Together — Example Page Layout

<header>
  <h1>Daily News</h1>
  <nav>
    <a href="#home">Home</a> |
    <a href="#world">World</a> |
    <a href="#sports">Sports</a> |
    <a href="#contact">Contact</a>
  </nav>
</header>

<main>
  <section>
    <article>
      <h2>Technology Advances in 2025</h2>
      <p>This year has seen amazing progress in artificial intelligence and web development tools.</p>
      <figure>
        <img src="tech-news.jpg" alt="Tech gadgets" width="400">
        <figcaption>New technology shaping our world.</figcaption>
      </figure>
    </article>
  </section>
  <aside>
    <h3>Did you know?</h3>
    <p>Semantic HTML helps with accessibility and SEO!</p>
  </aside>
</main>

<footer>
  <p>&copy; 2025 Daily News</p>
</footer>

Practice Section

Try these exercises:

  • Create a basic website layout using <header>, <main>, and <footer>.
  • Add a <nav> element inside the header with at least 3 links.
  • Inside <main>, add one <section> with two <article> elements about your favorite topics.
  • Add an <aside> section with one fun fact related to your article content.
  • Include a <figure> with an image and <figcaption> that explains it.

Summary

In this lesson, you learned how semantic HTML makes web content more organized and meaningful. Using these tags helps users and search engines understand your page better. Always choose the most meaningful tag for your content!
